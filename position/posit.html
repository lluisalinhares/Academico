Position

A propriedade position do CSS define a posição de um elemento em um documento. Para tal, esta propriedade funciona em conjunto com as propriedades left, right, top, bottom e z-index para determinar a posição final de um elemento em uma página.

Existem cinco valores que a propriedade position pode receber. Elas são:

  1. static
  2. relative
  3. absolute
  4. fixed
  5. sticky


Static

Valor padrão para qualquer elemento da página. Garantirá que ele seguirá o fluxo de renderização padrão atribuído pela propriedade display.

Absolute

Já esta propriedade, é posicionada em relação ao ancestral mais próximo, ou seja, permite que você posicione um elemento de acordo com o elemento pai (desde que este tenha um posicionamento diferente de static). Caso não haja pai, ele é colocado em relação ao inicial containing block. Sua posição final é determinada pelos valores de top, right, bottom, e left.

Podemos observar que o elemento é removido do fluxo normal de documentos e nenhum espaço é criado para ele no layout da página. Assim, outros elementos são posicionados como se ele não existisse.

Se um ancestral posicionado não existir, ele é posicionado em relação ao bloco de contenção do elemento raiz do documento.

Este valor cria um novo contexto de empilhamento quando o valor de z-indexnão é auto. As margens das caixas posicionadas absolutamente não colapsam com outras margens.

É importante comentar que quando utilizamos a posição absoluta, os elementos que originam esta posição passam a renderizar em uma camada acima de outros elementos que são do tipo static ou relativo, sobrepondo-se a elementos que não possuem posição ou z-index definido.

Maior parte do tempo, um elemento de posicionamento absoluto que tem height e width configurado como auto são dimensionados de acordo com seu conteúdo interno. No entanto, elementos não substituídos e absolutamente posicionados podem ser feitos para preencher o espaço vertical disponível especificando ambos top e bottom e deixando height sem especificar (ou seja, auto). Da mesma forma, eles podem preencher o espaço horizontal disponível especificando lefte righte deixando widthcomo auto. Exceto para esses casos descritos abaixo (para elementos de posicionamento absoluto que preenchem o espaço disponível): Se ambos top são bottom definidos (tecnicamento não sendo auto), top prevalece. Se ambos left são right definidos, left prevalece quando direction é ltr(Direção da escrita Portuguesa, Japonês na horizontal, etc.) e right prevalece quando direction for configurado rtl(Padrão de escrita em idiomas como Persa, Arabe, Hebraico, etc.)


Fixed

Este tipo de posição se comporta muito semelhante ao position absoluto , com a única diferença que sua referência de posição é ao elemento window/body e não ao elemento pai no qual este está inserido.

Sua posição final é determinada pelos valores de Top, Right, Bottom e Left.

Assim, o elemento é removido do fluxo normal de documentos e nenhum espaço é criado para o elemento no layout da página. Ele está posicionado em relação ao bloco inicial containing block estabelecido pelo viewport (exceto quando um de seus ancestrais tiver uma propriedade transform, perspective ou filter definido como algo diferente de none).

Este recurso permite que criemos elementos fixos que sejam independentes da rolagem da página, permanecendo eles nas posições definidas.


Sticky

Pode ser pensado como um híbrido de posicionamento relativo e fixo. Um elemento posicionado como sticky é tratado como posicionado relativo até cruzar um limite especificado, em cujo ponto ele é tratado como fixo até atingir o limite de seu pai.

Você deve especificar um limite com pelo menos um de Top, Right, Bottom, ou Left para que o posicionamento fixo se comporte como esperado. Caso contrário, ele será indistinguível do posicionamento relativo.
