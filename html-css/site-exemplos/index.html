Tags semânticas para desenvolvimento com HTML


Para <codar> é preciso praticar, então, bora lá!

  • De acordo com o dicionário Priberam, semântica é a área da linguística que estuda o significado e a sua relação com o significante.. Em HTML, semântica diz respeito a utilizar tags que façam uma marcação atribuindo sentido a cada tipo de conteúdo, ao invés de tags que são genéricas ou que apenas modifiquem a apresentação/estilo dos conteúdos marcados. Como o HTML é responsável pela estrutura e marcação dos conteúdos, utilizar as tags adequadas é fundamental para manter boas práticas de desenvolvimento web.

Tags semânticas

Negrito

  • Com sentido: <strong>

Esta tag define para o navegador que o conteúdo marcado é importante; desta forma, leitores de telas para pessoas com deficiência visual, dão uma ênfase quando topam com esta tag.

  • Sem sentido: <b>

Esta tag apenas torna o texto em negrito, modificando apenas a apresentação visual.


Italico

  • Com sentido: <em>

  • Sem sentido: <i>

Ambas tags com apresentação igual, mas, da mesma forma como o negrito, por "detrás dos panos", existe diferença.


Outras Tags importantes

  • <code> : Para escrever códigos em fonte monoespaçada, que fica melhor para visualização:

  • Pre : Para renderizar qualquer modificação no código, como espaçamentos, identação e etc., (por este motivo, é interessante colocar a tag code dentro dela, pra que a identação seja renderizada)
